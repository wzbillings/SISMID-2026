Day 3: ODEs with deSolve

Alex Edwards

What this half-day is for

By the end of this block, you should be able to:

  1. Read a compartment diagram as a system of ordinary differential equations.
  2. Translate the equations into the function shape required by deSolve.
  3. Solve the model with deSolve::ode().
  4. Plot and sanity-check simulated epidemic trajectories.

The goal is practical fluency: enough structure to read, modify, and run a basic infectious disease model in R.

The deSolve workflow

deSolve::ode() needs four things from you:

  1. y: the starting state of the system.
  2. times: the time points where you want results.
  3. func: a function that calculates the derivatives.
  4. parms: the model parameters.
deSolve::ode(
  y = initial_state,
  times = time_points,
  func = derivative_function,
  parms = parameters
)

Compartment models

Compartment models divide a population into states.

Source: DOI 10.3390/sym14122583

Compartment models: equations

For a simple SIR model:

\[ \begin{aligned} \frac{dS}{dt} &= -\beta S I \\ \frac{dI}{dt} &= \beta S I - \gamma I \\ \frac{dR}{dt} &= \gamma I \end{aligned} \]

The equations say how quickly people move between compartments.

Step 1: parameters

For this example:

  • beta: transmission rate.
  • gamma: recovery rate.
sir_parameters <- c(
  beta = 0.0015,
  gamma = 0.20
)

Step 2: starting state and time

The starting state is a named vector with one value per compartment.

sir_initial_state <- c(
  S = 999,
  I = 1,
  R = 0
)

sir_times <- seq(0, 80, by = 0.25)

The names matter. They become the column names in the solution.

Step 3: derivative function

The function signature is always:

function(t, state, parameters) {
  ...
}

Inside the function, calculate one derivative per state variable.

Step 3: derivative function, continued

sir_ode <- function(t, state, parameters) {
  S <- state["S"]
  I <- state["I"]
  R <- state["R"]

  beta <- parameters["beta"]
  gamma <- parameters["gamma"]

  dS <- -beta * S * I
  dI <- beta * S * I - gamma * I
  dR <- gamma * I

  list(c(S = dS, I = dI, R = dR))
}

Step 4: solve

sir_solution <- deSolve::ode(
  y = sir_initial_state,
  times = sir_times,
  func = sir_ode,
  parms = sir_parameters
)

Step 5: plot

sir_colors <- c(S = "deepskyblue3", I = "darkorange2", R = "seagreen4")

Sanity checks

Before trusting a simulation, ask:

  • Are all compartments non-negative?
  • Does the total population stay constant if the model has no births or deaths?
  • Does the direction of movement match the diagram?
  • Does changing one parameter move the curve in the direction you expect?

Sanity checks: code

Practice checkpoint: SIRS

Prompt

Modify the SIR model so recovered people lose immunity and become susceptible again. Call the new parameter zeta.

Source: DOI 10.3390/sym14122583 with course edit

Practice checkpoint: SIRS setup

Use:

sirs_parameters <- c(beta = 0.0015, gamma = 0.20, zeta = 0.02)
sirs_initial_state <- c(S = 999, I = 1, R = 0)
sirs_times <- seq(0, 160, by = 0.25)

Plot the result and compare it with the SIR curve.

Practice checkpoint: hint

The SIRS equations are:

\[ \begin{aligned} \frac{dS}{dt} &= -\beta S I + \zeta R \\ \frac{dI}{dt} &= \beta S I - \gamma I \\ \frac{dR}{dt} &= \gamma I - \zeta R \end{aligned} \]

You only need to change two things:

  1. Add zeta <- parameters["zeta"].
  2. Add the zeta * R flow into dS and out of dR.

Practice checkpoint: solution setup

sirs_parameters <- c(beta = 0.0015, gamma = 0.20, zeta = 0.02)
sirs_initial_state <- c(S = 999, I = 1, R = 0)
sirs_times <- seq(0, 160, by = 0.25)

Practice checkpoint: solution derivative

sirs_ode <- function(t, state, parameters) {
  S <- state["S"]
  I <- state["I"]
  R <- state["R"]

  beta <- parameters["beta"]
  gamma <- parameters["gamma"]
  zeta <- parameters["zeta"]

  dS <- -beta * S * I + zeta * R
  dI <- beta * S * I - gamma * I
  dR <- gamma * I - zeta * R

  list(c(S = dS, I = dI, R = dR))
}

Practice checkpoint: solution solve

sirs_solution <- deSolve::ode(

  times = sirs_times,
  func = sirs_ode,
  parms = sirs_parameters
)

Optional stretch: environmental pathogen

Prompt

For a faster student or take-home extension, add an environmental pathogen compartment P.

Source: Andreas Handel’s DSAIDE package

Optional stretch: equations

Use these equations:

\[ \begin{aligned} \frac{dS}{dt} &= n - b_I I S - b_P P S - mS \\ \frac{dI}{dt} &= b_I I S + b_P P S - gI - mI \\ \frac{dR}{dt} &= gI - mR \\ \frac{dP}{dt} &= qI - cP \end{aligned} \]

Optional stretch: implementation hints

Hint

Add a fourth state named P. Your derivative function should return four values: dS, dI, dR, and dP.

Solution outline

Start from sirs_ode(), add P <- state["P"], add parameters named n, b_I, b_P, m, g, q, and c, then return:

list(c(S = dS, I = dI, R = dR, P = dP))

Set n = 0 and m = 0 for the first run if you want to focus only on the environmental transmission path.

What to keep

The reusable pattern is:

  1. Name every state variable.
  2. Name every parameter.
  3. Write one derivative per state variable.
  4. Return the derivatives as list(c(...)).
  5. Plot first, then interpret.
