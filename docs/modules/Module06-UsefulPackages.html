Why these packages?

  • Every analysis ends with a table, a report, or a slide someone else has to read
  • Base R output (summary(), str()) is fast to write but ugly to share
  • These packages sit on top of your normal analysis and turn it into something you can paste into a paper, a report, or an email

Which packages?

  • here, rio, janitor → find, load, and clean your data
  • table1 → describe your sample
  • gt, gtsummary, sjPlot → build the tables people actually read
  • likert → visualize ordered/scale data
  • pacman → glue that loads everything with one line

About the Dataset

Every example in this deck uses the same real, de-identified file: QCRC_FINAL_Deidentified.xlsx

  • 288 patients admitted to the ICU with COVID-19
  • 10 sheets — Main_Dataset (demographics, treatments, outcomes), daily SOFA organ-failure scores, Long_Labs, Long_Vitals, ventilator settings, medications, and more
  • We’ll mainly use Main_Dataset: 37 variables including age, sex, race, BMI, intubation, CRRT/dialysis, and mortality
  • 98 of 288 patients (34%) died — a natural grouping variable for every table in this lecture

Before running this deck live: save the workbook as data/QCRC_FINAL_Deidentified.xlsx inside your project folder — the here() and rio::import() calls later assume that path.

Setup: pacman::p_load()

The problem with library()

A typical analysis script starts like this:

library(here)
library(rio)
library(janitor)
library(table1)
library(gt)
library(gtsummary)
library(sjPlot)
library(likert)
library(tidyverse)

If one of these isn’t installed, the whole script stops with an error — and you have to go install it manually, then re-run.

pacman::p_load()

pacman is a package-management package. Its star function, p_load(), installs anything missing, then loads everything — in one line.

# install.packages("pacman")   
# only needed once

pacman::p_load(
  here, rio, janitor, table1,
  gt, gtsummary, sjPlot, likert,
  tidyverse
)

pacman::p_load()

  • Checks each package name
  • Installs it from CRAN only if missing
  • Loads (library()s) all of them
  • Great for shared scripts, teaching, and reproducibility setups

here: Project-Relative Paths

The problem with setwd()

# fragile — breaks the moment this script runs on
# a different machine, or is Source()'d vs rendered
setwd("~/Downloads/SISMID2026")
qcrc <- import("data/QCRC_FINAL_Deidentified.xlsx")

here::here()

library(here)

#This is where it think we are
here()

#This gets us to our data
here("data", "QCRC_FINAL_Deidentified.xlsx")

#How do we read in data
qcrcHere <- import(here("data", "QCRC_FINAL_Deidentified.xlsx"))

here::here()

  • Finds your project root automatically (looks for a .Rproj file, .git folder, or similar) and builds paths from there
  • The same here() call works identically whether you run it from RStudio, the terminal, or a rendered Quarto document
  • Pairs perfectly with rio::import() — see next section

Rule of thumb: replace every setwd() and every hard-coded absolute path with here().

rio: Import & Export Anything

One function in, one function out

rio wraps dozens of format-specific packages (haven, readxl, data.table, jsonlite, …) behind two simple verbs.

library(rio)

  here("data", "QCRC_FINAL_Deidentified.xlsx"),
  sheet = "Main_Dataset"
)

dim(qcrc)
[1] 288  37

One function in, one function out

No more remembering read_csv() vs read.spss() vs read_excel() — and no hard-coded paths, thanks to here().

Exporting is just as easy

export(qcrc, here("data", "qcrc_clean.xlsx"))
export(qcrc, here("data", "qcrc_clean.rds"))
export(qcrc, here("data", "qcrc_clean.csv"))

clean_names()

  • Converts everything to consistent snake_case
  • Strips spaces and punctuation (even the colon), collapses repeated separators, and prefixes names that start with a digit (30D_... → x30d_...)
  • One line, run right after import() — every variable name used for the rest of this lecture assumes this has already happened

Bonus: fixing the "." missing code

Like a lot of clinical exports, this dataset stores missing values as a literal "." string — which leaves numeric columns like bmi, wbc, and crp stuck as text.

Any data frame → a polished table

  • gt() turns a data frame into a table object
  • Everything after is a layer: header, formatting, labels, footnotes, spanners, styling — piped together

Why gt matters

  • It’s the rendering engine many other packages (including gtsummary) build on top of
  • Output looks the same in HTML, PDF, Word, and RTF
  • Useful layers to know:
    • tab_spanner() — group columns under a shared header
    • tab_source_note() — add a footnote/citation
    • gtsave(tbl, "table.png") — export to image, HTML, RTF, or Word

tbl_summary() — descriptive tables

  • Median (IQR) instead of mean (SD) — a better default for skewed labs like CRP and WBC
  • Same goal as table1, but the output is already a polished gt table — ready to drop into a report
  • add_p() adds group-comparison p-values; add_overall() adds a combined column

tbl_regression() — model tables

  • Turns any lm/glm/coxph/mixed model into a clean estimate table (odds ratio, CI, p-value) — exponentiate = TRUE reports odds ratios instead of log-odds
  • add_glance_source_note() appends model fit stats (AIC, n)
  • tbl_merge() and tbl_stack() combine multiple models/tables side by side or on top of each other — handy for multi-model comparisons

gtsummary is the “batteries included” option — sensible defaults, statistical tests built in, minimal code. This is the same died ~ age + bmi + intubated + crrt model sjPlot will show next, just formatted differently.

likert: Ordinal / Scale Data

Beyond survey questions

The likert package is built for survey agreement scales, but it works for any ordered categorical variable measured on a shared scale. The SOFA sheet has six organ-system subscores, each rated 0 (normal) to 4 (severe failure) — structurally identical to a 5-point Likert item.

sofa <- import(
  here("data", "QCRC_FINAL_Deidentified.xlsx"),
  sheet = "SOFA"
) |>
  clean_names()

lvls <- as.character(0:4)

sofa_items <- sofa |>
  transmute(
    Coagulation    = factor(coag,  levels = lvls),
    Liver          = factor(liver, levels = lvls),
    CNS            = factor(cns,   levels = lvls),
    Renal          = factor(renal, levels = lvls),
    Cardiovascular = factor(cvs,   levels = lvls),
    Respiratory    = factor(resp,  levels = lvls)
  )

Putting It Together
