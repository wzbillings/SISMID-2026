A small manuscript problem

We are going to answer one concrete question with the measles data:

In Pakistan since 2000, which years should we flag for follow-up because first-dose measles vaccine coverage was low and measles incidence was elevated?

A small manuscript problem: manuscript arc

This is a miniature version of the Exercise 1 manuscript arc:

  1. State a question.
  2. Translate the question into a data rule.
  3. Run the rule.
  4. Write a cautious result.

The programming idea

We can describe the rule in plain English:

If MCV1 coverage is below 90% and measles incidence is at least 1 case per 100,000 people, call it a priority year.

Now the problem is not “Can I memorize R?”

The problem is:

How do I turn that sentence into computer code?

What do we need?

For this question, we need:

  • country, because we are starting with Pakistan
  • year, because we only want 2000 and later
  • measles_cases, because we need incidence
  • population, because raw case counts are hard to compare
  • MCV1_coverage, because that is our coverage variable

Step 3: translate each condition

The English rule has two parts:

  1. MCV1 coverage is below 90%.
  2. Measles incidence is at least 1 case per 100,000 people.

In R, each part becomes a logical vector.

low_coverage <- pakistan$MCV1_coverage < 90
elevated_incidence <- pakistan$cases_per_100k >= 1

Missing values matter

If coverage is missing, R cannot decide whether coverage is below 90%.

table(low_coverage, useNA = "ifany")
low_coverage
TRUE <NA> 
  23   25

That is not a failure. It is R refusing to guess.

For a manuscript-style analysis, we need to decide what we want to do with missing information.

Step 4: write the decision rule

Here is the rule in full:

  1. If coverage or incidence is missing, label the row "missing information".
  2. Otherwise, if coverage is below 90% and incidence is at least 1, label the row "priority year".
  3. Otherwise, label the row "not priority".

Step 4: code the decision rule

pakistan$priority_year <- ifelse(
  is.na(pakistan$MCV1_coverage) | is.na(pakistan$cases_per_100k),
  "missing information",
  ifelse(
    pakistan$MCV1_coverage < 90 & pakistan$cases_per_100k >= 1,
    "priority year",
    "not priority"
  )
)

What did ifelse() do?

ifelse() has three pieces:

ifelse(test, value_if_true, value_if_false)

Our nested version says:

  • first check whether the data are missing
  • if not, check whether the year meets the priority rule
  • give every row one label

Student prompt

Change the question:

In Afghanistan since 2010, which years should be flagged if MCV2 coverage was below 80% or measles incidence was at least 5 cases per 100,000 people?

Write the code before looking at the solution.

Active checkpoint: hint

You need to change four pieces:

  1. the country
  2. the start year
  3. the coverage variable and cutoff
  4. the logical connector, because this rule uses “or”

The R operator for “or” is |.

Active checkpoint: solution

afghanistan <- measles[
  measles$country == "Afghanistan" & measles$year >= 2010,
]

afghanistan <- afghanistan[order(afghanistan$year), ]
afghanistan$cases_per_100k <- afghanistan$measles_cases / afghanistan$population * 100000

Active checkpoint: solution rule

afghanistan$priority_year <- ifelse(
  (!is.na(afghanistan$MCV2_coverage) & afghanistan$MCV2_coverage < 80) |
    (!is.na(afghanistan$cases_per_100k) & afghanistan$cases_per_100k >= 5),
  "priority year",
  ifelse(
    is.na(afghanistan$MCV2_coverage) | is.na(afghanistan$cases_per_100k),
    "missing information",
    "not priority"
  )
)
