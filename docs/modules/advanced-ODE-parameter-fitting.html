Reuse the SIR model

sir_ode <- function(t, state, parameters) {
  S <- state["S"]
  I <- state["I"]
  R <- state["R"]

  beta <- parameters["beta"]
  gamma <- parameters["gamma"]

  dS <- -beta * S * I
  dI <- beta * S * I - gamma * I
  dR <- gamma * I

  list(c(S = dS, I = dI, R = dR))
}

Write a simulation helper

simulate_sir <- function(beta, gamma, times) {
  deSolve::ode(
    y = c(S = 999, I = 1, R = 0),
    times = times,
    func = sir_ode,
    parms = c(beta = beta, gamma = gamma)
  )
}

Objective function

We need one number that says how badly a parameter pair fits.

Here we use the sum of squared residuals between observed infected counts and the simulated infected compartment:

\[ SSR = \sum_i \left(I_i - \hat{I}_i(\beta, \gamma)\right)^2 \]

sir_ssr <- function(par) {
  beta <- par[1]
  gamma <- par[2]

  if (beta <= 0 || gamma <= 0) {
    return(1e12)
  }

  simulated <- tryCatch(
    simulate_sir(
      beta = beta,
      gamma = gamma,
      times = observed_sir$day
    ),
    error = function(e) NULL
  )

  if (is.null(simulated)) {
    return(1e12)
  }

  predicted_infected <- simulated[, "I"]

  if (any(!is.finite(predicted_infected))) {
    return(1e12)
  }

  fit <- sum((observed_sir$total_cases - predicted_infected)^2)

  if (!is.finite(fit)) {
    return(1e12)
  }

  fit
}

Practice checkpoint

Prompt

Change the starting values in optim() to beta = 0.005 and gamma = 0.50. Do the fitted parameters return to approximately the same values?

Hint

Only change the par = ... line. Keep the same bounds.

Solution outline

Run the same call with:

par = c(beta = 0.005, gamma = 0.50)

For this small example, the bounds and data are informative enough that the result should be very similar. For more complex ODE systems, starting values can matter a lot.

Take-home cautions

  • Real case data are noisy, delayed, underreported, and often only partly observed.
  • The observation model matters as much as the ODE model.
  • Parameter uncertainty from this quick example is not reliable.
  • Use this deck to learn how to connect simulation code to a fitting workflow, then reach for a more serious modeling framework for real inference.
